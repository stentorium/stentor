# Documentation Scripts

## fix-api-docs.js

Post-processes API documentation generated by Microsoft API Documenter to fix MDX/JSX compatibility issues.

### Problem

API Documenter generates markdown files with XML tags (like `<item>`, `<enclosure>`, etc.) which Docusaurus's MDX parser interprets as JSX components, causing build failures.

### Solution

This script automatically escapes problematic XML tags as inline code:
- `<item>` becomes `` `<item>` ``
- `<enclosure>` becomes `` `<enclosure>` ``

### Usage

The script runs automatically as part of the `yarn docs` command:

```bash
# Manually run
node scripts/fix-api-docs.js

# Or as part of docs generation
yarn docs
```

### Problematic Tags

The script handles these common XML/RSS tags:
- `<item>`, `<enclosure>`, `<title>`, `<description>`
- `<link>`, `<pubDate>`, `<guid>`, `<channel>`
- `<rss>`, `<content>`, `<author>`, `<category>`
- `<comments>`, `<source>`

### Integration

Automatically runs after API documentation generation:
1. `api-documenter` generates markdown files
2. This script escapes problematic tags
3. Docusaurus can safely build the docs
